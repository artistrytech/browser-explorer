# Explorer ベース ファイル管理 + 編集 + Git アプリ 設計書 (v2)

## 0. ドキュメントについて

本書は、**Explorer（ファイル管理）を主画面**とし、そこにテキスト編集と Git 管理を統合した
Web アプリの設計書である。ブラウザで動作するが単一アプリであり、デスクトップ環境
（ウィンドウマネージャ）ではない。動作環境は **Mac** を想定し、
**Mac を使う Windows ユーザーが違和感なく使えること**を最優先とする。

搭載機能は次の 3 本柱。

- **Explorer**（ファイル管理・主画面）
- **テキストエディタ**（サクラエディタ相当・Explorer 内に統合）
- **Git クライアント**（Tortoise Git 相当・Explorer 内に統合）

---

## 1. 概要

### 1.1 目的とコンセプト

- Mac に乗り換えた／Mac を併用する Windows ユーザーに、**Windows Explorer の操作感**をそのまま提供する。
- ファイル管理を主軸に、その延長線上で**編集**と**Git 操作**を完結させる。
- 実ファイルと直結し、Finder の代わりに日常のファイル操作を担えるツールを目指す。

### 1.2 「Windows ユーザーが Mac で快適」の具体像

Mac（Finder）と Windows（Explorer）で操作が食い違い、Windows ユーザーがつまずく代表例を、
本アプリでは **Windows 流に寄せる**。ここが本アプリの存在価値の中心。

| 操作 | Finder（Mac 標準） | Windows Explorer | 本アプリの方針 |
| --- | --- | --- | --- |
| リネーム | Return | **F2** | F2 に対応（Return は開く） |
| 開く | Space=プレビュー / ⌘↓ | **Return / ダブルクリック** | Return で開く |
| 上の階層へ | ⌘↑ | **Alt+↑** | Alt+↑ に対応 |
| 戻る / 進む | ⌘[ / ⌘] | **Alt+← / Alt+→** | Alt+← / Alt+→（ブラウザ履歴と連動） |
| 削除 | ⌘Delete | **Delete** | Delete でゴミ箱へ |
| 切り取り移動 | ⌘C→⌘⌥V（分かりにくい） | **Ctrl+X→Ctrl+V** | 切り取り→貼り付けで移動を再現 |
| コピー/貼付 | ⌘C / ⌘V | Ctrl+C / Ctrl+V | 同等操作を提供 |
| 拡張子表示 | 既定で一部非表示 | 表示 | 既定で拡張子を常時表示 |
| アドレス表示 | パス非表示寄り | パス/パンくず | アドレスバー + パンくずを常設 |
| 表示 | アイコン中心 | 詳細（列）表示 | 「詳細」表示を既定に |

### 1.3 前提・スコープ

| 項目 | 方針 |
| --- | --- |
| ファイルシステム | 仮想ではなく実ファイル連動。バックエンド Node.js が実操作 |
| 実行形態 | ローカルサーバー前提。`npm run dev` で起動しブラウザで開く |
| 対象 OS | **macOS**（Unix パス、`/` 区切り、マウントは `/Volumes/*`） |
| 利用者 | 個人・単一ユーザー。認証なし |
| 操作範囲 | **全ドライブ（ファイルシステム全体）許可**（後述のガードは入れる） |
| エディタ | **Monaco Editor** に確定 |
| Git | **システムにインストール済みの git のラッパー**。認証は git 側に委譲 |
| 永続化 | **サーバー側 DB（SQLite）**。設定/状態のエクスポート・インポート対応 |
| キーバインド | **Ctrl 基準 / ⌘ 基準を設定で切替可能**（既定は Ctrl 基準） |
| ナビゲーション | **カレントパスは URL に埋め込む**。未指定時はルート `/`。戻る/進むはブラウザ履歴と連動 |
| ワークスペース概念 | **なし**。「よく使う場所」はブックマーク（クイックアクセス）のみで表現 |
| スコープ外 | ターミナル、フォルダ横断の全文検索/Grep など（初期は含めない） |

## 2. セキュリティ方針（全ドライブ許可の前提で）

「セキュリティは緩くて良い」「全ドライブにしたい」を受けたうえでの整理。
**機能的には全ドライブで問題ないが、ローカルサーバー特有の 1 点だけは対策する。**

### 2.1 唯一気にすべきリスク

このアプリの API は「ファイルシステム全体を読み書き・削除できる」極めて強い権限を持つ。
localhost で待ち受けるサーバーは、**利用者がブラウザで開いている別サイト**から到達され得る
（CSRF / DNS リバインディング）。無防備だと、悪意あるページが localhost 宛に破壊的リクエストを
送ったり、`~/.ssh/id_rsa` 等の機微ファイルを読ませて外部送信する、という攻撃が理論上成立する。

### 2.2 対策（安価かつ十分）

以下だけで上記の穴はほぼ塞げる。個人利用でもこれは入れておく。

1. **127.0.0.1 のみで待ち受け**（`0.0.0.0` にしない。LAN に出さない）。
2. **`Origin` / `Host` ヘッダ検証**：アプリ自身のオリジン以外からの API 呼び出しを拒否
   → DNS リバインディングとクロスサイトからの操作を無効化。
3. **セッショントークン必須**：配信するフロントに埋め込んだトークンを API 呼び出しに要求
   → 他サイトからのブラインド CSRF を無効化。
4. **削除は既定でゴミ箱送り**（macOS のゴミ箱）。完全削除は明示操作のみ。

### 2.3 任意の追加ガード

- ごく機微なパス（`~/.ssh`、`~/Library/Keychains` など）は既定で「隠し + 操作時に確認」。
  （ワークスペースの概念は持たないため、ワークスペース外を検知する類のガードは設けない。）

> 結論: 上記 2.2 を入れたうえで全ドライブ許可で運用可。認証・サンドボックス等の重い仕組みは不要。

## 3. 全体アーキテクチャ

### 3.1 構成図

```
┌──────────────────────────────────────────────┐
│               ブラウザ (単一アプリ)            │
│  ┌────────────────────────────────────────┐  │
│  │  Explorer 主画面                         │  │
│  │  ┌──────────┬──────────────────────────┐│  │
│  │  │ Sidebar  │ File一覧 / Editorタブ /   ││  │
│  │  │ (ツリー) │ Git パネル(切替・分割)    ││  │
│  │  └──────────┴──────────────────────────┘│  │
│  └────────────────────────────────────────┘  │
│          │ REST(fetch)      │ WebSocket        │
└──────────┼───────────────────┼─────────────────┘
           ▼                   ▼
┌──────────────────────────────────────────────┐
│           ローカルサーバー (Node.js, Mac)      │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐  │
│  │FS Svc  │ │Git Svc │ │Watch WS│ │State DB│  │
│  └───┬────┘ └───┬────┘ └───┬────┘ └───┬────┘  │
└──────┼──────────┼──────────┼──────────┼────────┘
       ▼          ▼          ▼          ▼

       └──── 実ファイルシステム(/, /Volumes) ────┘
```

- **単一ウィンドウ**。Explorer が土台で、編集・Git はその中の「タブ / パネル」として現れる。
- サーバーが実ファイル・git・監視・状態永続化を担う唯一の窓口。
- 通信は REST（同期操作）＋ WebSocket（ファイル変更・Git 状態のプッシュ）。

### 3.2 技術スタック

| レイヤ | 採用 | 補足 |
| --- | --- | --- |
| Frontend | React + TypeScript + Vite | 状態管理 Zustand |

| エディタ | **Monaco Editor** | 確定。検索置換/複数カーソル/ハイライト等を標準装備 |

| ゴミ箱 | `trash` | macOS ゴミ箱へ |
| 永続化 | **SQLite（`better-sqlite3`）** | 単一ユーザー・ファイル DB |

├──────────────┬────────────────────────────────────────────┤
│ クイックアクセス│ [ 一覧 ] または [ editor: index.ts × | … ] │
│  ★ Home       │  名前         種類   サイズ  更新日時       │
│  ★ project    │  📁 src       Folder  --    2026/..        │

- **ツールバー**: 戻る/進む/上へ、アドレスバー（パンくず＋直接入力）、検索、設定。

> 主画面は 1 枚。「別ウィンドウが大量に開く」体験ではなく、VS Code のように
> 1 ウィンドウ内でタブ/パネルを切り替える。

## 5. バックエンド API 設計

REST 基本、変更通知のみ WebSocket。すべて `/api` 配下。§2.2 のトークン/Origin 検証を全 API に適用。

### 5.1 ファイルシステム API

| メソッド | エンドポイント | 説明 |
| --- | --- | --- |
| GET | `/api/fs/volumes` | ルート/マウント一覧（`/`, `/Volumes/*`） |
| GET | `/api/fs/list?path=` | ディレクトリ内容（名前/種別/サイズ/更新日時/属性/シンボリックリンク） |
| GET | `/api/fs/stat?path=` | 単一エントリ詳細（プロパティ用） |
| GET | `/api/fs/read?path=&encoding=` | 読み込み（省略時は自動判定、元エンコ/改行も返す） |
| POST | `/api/fs/write` | 書き込み（path, content, encoding, eol, bom） |
| POST | `/api/fs/mkdir` | フォルダ作成 |
| POST | `/api/fs/create` | 空ファイル作成 |
| POST | `/api/fs/rename` | 名前変更 |
| POST | `/api/fs/move` | 移動（src[], destDir） |
| POST | `/api/fs/copy` | コピー（src[], destDir） |
| DELETE | `/api/fs/delete` | 削除（既定ゴミ箱、`permanent=true` で完全削除） |
| GET | `/api/fs/search?dir=&query=` | カレント配下の名前検索 |

**文字コード / 改行（サクラ要件）**

- 読み込み: `jschardet` 判定 → `iconv-lite` で UTF-8 化して返却。元エンコ・改行・BOM 有無も返す。
- 書き込み: 指定エンコ（UTF-8 / Shift-JIS / EUC-JP / UTF-16 …）・改行（CRLF/LF/CR）・BOM で書き戻す。

### 5.2 Git API（システム git のラッパー）

`simple-git` 経由で**利用者環境の git を実行するだけ**。
認証は git の仕組み（macOS Keychain の credential helper、SSH agent 等）に委譲し、
**アプリはトークンや鍵を一切保持・管理しない**。

| メソッド | エンドポイント | 説明 |
| --- | --- | --- |
| GET | `/api/git/is-repo?path=` | Git 管理下か判定 |
| GET | `/api/git/status?repo=` | 変更一覧（staged/unstaged/untracked） |
| GET | `/api/git/log?repo=&limit=` | 履歴（グラフ用の親子情報含む） |
| GET | `/api/git/diff?repo=&path=&staged=` | 差分 |
| POST | `/api/git/stage` `/unstage` `/discard` | ステージ/解除/破棄 |
| POST | `/api/git/commit` | コミット（message, amend?） |
| POST | `/api/git/push` `/pull` `/fetch` | 同期 |
| GET | `/api/git/branches?repo=` | ブランチ一覧 |
| POST | `/api/git/branch` | 作成/切替/削除 |
| POST | `/api/git/merge` | マージ |
| POST | `/api/git/stash` | save/pop/list |
| GET | `/api/git/blame?repo=&path=` | blame |
| POST | `/api/git/init` `/clone` | init / clone |

> **認証の注意**: サーバーは非対話で git を実行するため、push/pull が資格情報を対話要求すると
> 失敗する。Mac 既定の `osxkeychain` helper か SSH agent が構成済みであることを前提とし、
> 未構成時は「ターミナルで一度 git 認証を通してください」と案内する（アプリ側で認証はしない）。

### 5.3 状態永続化 API（サーバー DB）

| メソッド | エンドポイント | 説明 |
| --- | --- | --- |
| GET | `/api/state` | 保存済み状態の取得（設定・レイアウト・お気に入り等） |
| PUT | `/api/state` | 状態の保存（部分更新可） |
| GET | `/api/state/export` | 全状態を JSON でエクスポート（ダウンロード） |
| POST | `/api/state/import` | JSON をインポートして復元 |

### 5.4 WebSocket（変更通知）

| イベント | ペイロード | 用途 |
| --- | --- | --- |
| `fs:change` | `{ type, path }` | 一覧/エディタの自動更新 |
| `git:status` | `{ repo, dirtyFiles }` | オーバーレイアイコン更新 |

## 6. 機能①：Explorer（主画面）

### 6.1 ナビゲーション

- サイドバーのツリー展開/折りたたみ、フォルダのダブルクリック/Return で移動。
- **上の階層へ: Alt+↑**、**戻る: Alt+←**、**進む: Alt+→**（戻る/進むはブラウザ履歴と連動）。
- アドレスバー: パンくず表示、クリックで直接パス入力に切替。
- クイックアクセス（ピン留めフォルダ = ブックマーク）の登録・並べ替え（→ DB 永続化）。
  ※ ワークスペースの概念は持たず、「よく使う場所」はこのブックマークで表現する。

### 6.2 表示

- 表示モード: 詳細（既定・列: 名前/種類/サイズ/更新日時）/一覧/大アイコン。
- 各列ソート、隠しファイル表示切替、**拡張子は常時表示**。
- 拡張子ごとのアイコン割り当て。

### 6.3 ファイル操作（Windows 流キー）

- 新規フォルダ / 新規ファイル。
- リネーム（**F2**）、開く（**Return / ダブルクリック**）。
- コピー/切り取り/貼り付け（**Ctrl+C / Ctrl+X / Ctrl+V**、切り取り→貼付で移動）。
- 削除（**Delete**=ゴミ箱、**Shift+Delete**=完全削除の確認付き）。
- 複数選択（Ctrl / Shift クリック・範囲選択）、ドラッグ＆ドロップ移動/コピー。
- プロパティ（サイズ・パス・更新日時・属性・リンク先）。
- 右クリックのコンテキストメニュー（上記操作 + 「エディタで開く」「Git 操作」）。

### 6.4 検索

- カレントフォルダ配下の名前検索（フォルダ横断の全文検索は初期スコープ外）。

### 6.5 URL とナビゲーション（状態は URL に埋め込む）

- **カレントパスを URL に埋め込む**。例:
  - `http://localhost:PORT/?path=/Users/me/project`
  - **`path` 未指定時はルート `/` を表示**。
- フォルダ移動のたびに URL を書き換える（`history.pushState`）。これにより:
  - **Alt+← / Alt+→（戻る/進む）= ブラウザ履歴の back/forward** にそのまま対応。
  - **Alt+↑（上の階層）= 親パスへ移動**（URL も親に更新）。
  - 特定フォルダを開いた状態を**ブックマーク/共有**でき、リロードしても同じ場所に戻る。
- 起動時は「前回セッションの自動復元」を行わず、**URL のパス（なければルート）**を初期表示とする。
- 補足: 開いていた**エディタタブ等のライブなセッション**は URL/DB で自動復元しない。
  永続化するのは設定・ブックマーク・最近使った項目・リポジトリ一覧など（§9）。

## 7. 機能②：テキストエディタ（サクラ相当・Monaco）

### 7.1 UI

- ファイルを開くとメイン領域が**タブ付きエディタ**に切替。一覧との分割表示も可。
- 上部にメニュー（ファイル/編集/検索/表示）、下部ステータスに行列・文字コード・改行・言語。

### 7.2 ファイル

- 新規/開く/上書き保存/名前を付けて保存/すべて保存。
- タブによる複数ファイル同時編集、Explorer からの「開く」連携。
- 未保存クローズ確認、外部変更（WS）検知 → 再読込ダイアログ。

### 7.3 編集（Monaco 標準を活用）

- Undo/Redo、切り取り/コピー/貼り付け、複数カーソル/矩形選択。
- 行の移動/複製/削除、自動インデント、タブ幅、タブ↔スペース変換、折りたたみ。

### 7.4 検索・置換（サクラの要）

- インクリメンタル検索、検索/置換ダイアログ（**正規表現**・大文字小文字・単語単位）。
- ファイル内一括置換、指定行ジャンプ。

### 7.5 表示

- シンタックスハイライト（拡張子で言語自動判定＋手動切替）、行番号、折り返し切替、
  空白/改行/タブ可視化、テーマ（ライト/ダーク）、フォント/サイズ（→ DB 永続化）。

### 7.6 文字コード・改行（重要要件）

- 開いた際に文字コード・改行を自動判定して表示。
- ステータスバーから**エンコーディング変更で再読込 / 変換保存**（UTF-8 / Shift-JIS / EUC-JP / UTF-16）。
- 改行切替（CRLF / LF / CR）、BOM 有無切替。

## 8. 機能③：Git クライアント（Tortoise Git 相当）

Tortoise Git の「エクスプローラー統合 + ダイアログ」体験を、Explorer 統合 + パネルで再現。

### 8.1 Explorer 統合：状態オーバーレイ

| 状態 | オーバーレイ | 意味 |
| --- | --- | --- |
| Normal | ✔ | 変更なし |
| Modified | ● | 変更あり |
| Added/Staged | ＋ | 追加/ステージ済み |
| Untracked | ？ | 未追跡 |
| Conflicted | ⚠ | 競合 |
| Ignored | － | 無視対象 |

- WS `git:status` を購読し、一覧アイコンに重畳。右クリックに Git 操作群を追加。
- サイドバー「リポジトリ」からリポジトリと現在ブランチを一覧。

### 8.2 コミットパネル

- 変更一覧のチェックでステージ/アンステージ、選択ファイルの差分プレビュー。
- メッセージ入力 → Commit / Commit & Push、amend 対応。

### 8.3 ログ / 履歴ビューア

- 履歴の**グラフ表示**（ブランチ/マージ線）、コミット詳細（作者/日時/メッセージ/変更ファイル）。
- コミット選択で差分、ファイル単位の履歴、reset/checkout（右クリック）。

### 8.4 差分ビューア

- side-by-side / inline、行ハイライト（追加=緑/削除=赤）。
- 作業ツリー vs HEAD、コミット間、ステージ vs HEAD に対応。

### 8.5 ブランチ / 同期 / その他

- ブランチ一覧・作成/切替/削除/マージ、fetch/pull/push、stash、blame、init/clone。
- 認証は §5.2 のとおり git 側に委譲（アプリは資格情報を扱わない）。

## 9. 状態の永続化・エクスポート（サーバー DB）

### 9.1 永続化先

- サーバー側 **SQLite**（`data/app.db`）。単一ユーザー・ファイル DB でバックアップも容易。

### 9.2 永続化する項目

| カテゴリ | 例 |
| --- | --- |
| クイックアクセス（ブックマーク） | ピン留めフォルダと並び順 |
| 表示設定 | 表示モード、ソート、隠しファイル、拡張子表示 |
| キーバインド | **Ctrl 基準 / ⌘ 基準の選択**（既定 Ctrl 基準） |
| エディタ設定 | テーマ、フォント/サイズ、タブ幅、既定エンコ/改行 |
| UI レイアウト | パネル幅（サイドバー/一覧/エディタの分割比） |
| リポジトリ | 登録済みリポジトリ一覧 |
| 最近使った項目 | 最近開いたファイル/フォルダ |

> **カレントパス・開いていたタブは永続化しない**（§6.5）。カレントパスは URL 駆動、
> タブはセッション内のみ保持。永続化するのは上表のような設定・ブックマーク類に限る。

### 9.3 テーブル設計（概略）

```sql
-- 汎用 key-value（設定・レイアウト）
CREATE TABLE settings (key TEXT PRIMARY KEY, value TEXT NOT NULL);

-- ピン留め
CREATE TABLE favorites (id INTEGER PRIMARY KEY, path TEXT, label TEXT, sort INTEGER);

-- リポジトリ
CREATE TABLE repositories (id INTEGER PRIMARY KEY, path TEXT UNIQUE, added_at TEXT);

-- 最近使った項目
CREATE TABLE recents (id INTEGER PRIMARY KEY, path TEXT, kind TEXT, opened_at TEXT);
```

### 9.4 エクスポート / インポート

- `GET /api/state/export`: 上記全テーブルを 1 つの JSON にまとめてダウンロード（移行/バックアップ用）。
- `POST /api/state/import`: JSON を読み込み、既存を置換または統合して復元。

| 項目 | 方針 |

## 12. 確定した設計判断（確認事項の回答反映）

| 論点 | 決定 |
| --- | --- |
| キーバインド基準 | **Ctrl 基準 / ⌘ 基準を設定で切替可能**（既定は Ctrl 基準） |
| ワークスペース概念 | **持たない**。「よく使う場所」はブックマーク（クイックアクセス）のみ |
| セッション/カレントパス | **URL にパスを埋め込む**。未指定時はルート `/`。戻る/進むはブラウザ履歴と連動 |
| Explorer ナビキー | **上へ = Alt+↑ / 戻る = Alt+← / 進む = Alt+→** |

現時点で未決の設計論点はなし。次段階では、フェーズ 1（主画面 + Explorer）から
画面ワイヤーフレームと FS API の入出力スキーマの詳細化に進むのが妥当。
